# AstrBot JMComic 禁漫搜索插件

从禁漫天堂搜索本子，支持分页结果（合并转发封面+名字）、查看详情、下载章节为 PDF。

基于 [JMComic-Crawler-Python](https://github.com/hect0x7/JMComic-Crawler-Python) 库实现。

## 安装

1. 将本插件文件夹放入 AstrBot 的 `data/plugins/` 目录
2. 安装依赖：
   ```bash
   pip install -r requirements.txt
   ```
3. 重启 AstrBot

## 配置

将 `config.yaml.example` 复制为 `config.yaml`，按需修改：

```yaml
client_impl: "api"   # "api"(推荐,不限IP) 或 "html"(需特定地区IP)
proxy: ""            # 代理，如 http://127.0.0.1:7890
domains: []          # 自定义域名，留空使用默认
```

## 命令速查

| 命令 | 说明 |
|------|------|
| `/jm <关键词>` | 搜索本子，8条/页，最多3页24条，合并转发封面+名字 |
| `+` 或 `下一页` | 翻到下一页 |
| `-` 或 `上一页` | 翻到上一页 |
| `/jm <序号>` | 有缓存：全局序号看详情；无缓存：禁漫ID查详情 |
| `/jm <本子ID>` | 直接按禁漫ID查看详情 |
| `/jm看图 <N>` | 下载全局第N本的第1话 → 拼接PDF发送 |
| `/jm看图 <N>` | 已查看详情后，下载第N话PDF（N=章节序号） |
| _(缓存5分钟自动过期)_ | 过期后再 /jm 数字 = 禁漫ID直查 |

## 完整使用演示

###： 1. 搜索本子

```
/jm 少女
```

Bot 返回合并转发消息，每页8个结果（封面图 + 名字），末尾显示：
```
「少女」第1/3页
/jm <序号>看详情  +下一页  -上一页  /jm看图 <序号>
```

### 2. 翻页浏览

```
+          （或发送 "下一页"）
-          （或发送 "上一页"）
```

### 3. 查看详情 — 回复全局序号

搜索结果编号是全局的（第1页1~8，第2页9~16，第3页17~24），直接回复数字即可：

```
23         ← 查看第23本详情
```

Bot 返回合并转发消息，包含：
- 封面图
- 本子名、ID、作者、标签
- 章节列表（第1话、第2话...）

### 4. 直接按禁漫ID查看

```
/jm 123456
```

跳过搜索，直接查看 JM123456 的详情。

### 5. 下载本子图片 — PDF

```
/jm看图 23    ← 下载第23本（全局序号）的第1话，生成PDF
```

或先查看详情后，按章节序号下载指定话：

```
/jm看图 3     ← 下载第3话（章节序号）
```

Bot 先压缩图片（JPEG quality 75，长边 1600px），再拼接为 PDF 发送。

```
共 20 张，第3话 xxx.pdf → [PDF文件]
```

### 6. 完整流程演示

```
用户: /jm 少女
Bot:  搜索「少女」，找到 20 个结果：
Bot:  [合并转发] 8本封面+名字，末尾 「少女」第1/3页

用户: +
Bot:  [合并转发] 8本封面+名字，末尾 「少女」第2/3页

用户: /jm 14
Bot:  [合并转发] 第14本详情（封面+文字+章节列表）

用户: /jm看图 14
Bot:  共 24 张，第14本 第1话 xxx.pdf
Bot:  [PDF文件]
```

## 依赖

- jmcomic >= 2.6.0
- img2pdf >= 0.4.4（推荐，流式图片转PDF）
- Pillow >= 9.0.0（img2pdf 不可用时备选）
- PyPDF2 >= 3.0.0（Pillow 模式下合并PDF分块）
